
## **JavaScript 继承方式全景图**

```
继承方式
│
├── 1. 原型链继承
│     ├─ 实现：Child.prototype = new Parent()
│     ├─ 优点：继承父类原型方法
│     └─ 缺点：
│           1. 无法传参
│           2. 引用类型共享
│
├── 2. 构造函数继承（借用构造函数）
│     ├─ 实现：Parent.call(this, args)
│     ├─ 优点：可以传参，避免引用类型共享
│     └─ 缺点：无法继承父类原型方法
│
├── 3. 组合继承（原型链 + 构造函数）
│     ├─ 实现：
│     │     Parent.call(this, args)   // 继承实例属性
│     │     Child.prototype = Object.create(Parent.prototype) // 继承原型方法
│     ├─ 优点：
│     │     1. 可传参
│     │     2. 可继承原型方法
│     └─ 缺点：父构造函数调用两次
│
└── 4. ES6 class 继承
      ├─ 实现：class Child extends Parent { constructor() { super(); } }
      ├─ 优点：
      │     1. 语法清晰
      │     2. 支持 super 调用父类构造函数
      └─ 面试点：
            - class 只是语法糖，底层仍是原型链 + 构造函数
            - super 用于调用父类构造函数或方法
```

---

### 🔹 关系总结

1. **原型链继承** → 继承方法
2. **构造函数继承** → 继承属性
3. **组合继承** → 属性 + 方法都继承（最常用 ES5 手写方式）
4. **ES6 class** → 语法糖封装组合继承，推荐实际开发使用
