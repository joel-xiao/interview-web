
## **OSI 七层 & TCP/IP 四层对比 + 浏览器请求流程图**

```
浏览器请求（HTTP/HTTPS）
       │
       ▼
┌───────────────────────────────┐
│      OSI 应用层 (7)           │
│  提供网络服务、HTTP/HTTPS     │
│  TCP/IP 对应：应用层          │
└───────────────────────────────┘
       │
       ▼
┌───────────────────────────────┐
│      OSI 表示层 (6)           │
│  数据编码/加密/解密           │
│  TLS/SSL 加密                 │
└───────────────────────────────┘
       │
       ▼
┌───────────────────────────────┐
│      OSI 会话层 (5)           │
│  会话建立、维护和终止         │
│  WebSocket 会话管理           │
└───────────────────────────────┘
       │
       ▼
┌───────────────────────────────┐
│      OSI 传输层 (4)           │
│  TCP/UDP，端到端通信，可靠传输│
│  TCP 三次握手 / 四次挥手       │
└───────────────────────────────┘
       │
       ▼
┌───────────────────────────────┐
│      OSI 网络层 (3)           │
│  IP 路由、寻址               │
│  数据包从源 IP 到目标 IP      │
└───────────────────────────────┘
       │
       ▼
┌───────────────────────────────┐
│      OSI 数据链路层 (2)       │
│  MAC 地址，帧传输             │
│  局域网通信、交换机           │
└───────────────────────────────┘
       │
       ▼
┌───────────────────────────────┐
│      OSI 物理层 (1)           │
│  电缆、光纤、信号传输         │
└───────────────────────────────┘
       │
       ▼
        服务器接收请求
```

---

### 🔹 图解说明

1. **OSI 七层**：理论分层，便于理解网络协议职责

2. **TCP/IP 四层**：实际应用更贴近浏览器通信

   * **应用层** = OSI 7+6+5
   * **传输层** = OSI 4
   * **网络层** = OSI 3
   * **链路+物理层** = OSI 2+1

3. **数据流**：

   * 浏览器 HTTP 请求 → TCP/IP 栈 → 物理介质 → 服务器 TCP/IP 栈 → HTTP 处理
   * 返回响应同理

4. **面试高频点**：

   * TCP 三次握手 / 四次挥手
   * HTTPS/TLS 握手
   * IP 与 MAC 区别
   * HTTP/HTTPS 工作层级
